# Задача struct: Мёртвые нули

Павел Иванович Чичиков весьма доволен компьютерными технологиями: теперь вместо множества бумажных списков он может собирать сведения о крестьянах на свою любимую дискету объёмом `4` килобайта (`4096` байтов). За каждую заполненную дискету он платит Плюшкину 100 рублей.

Степан Плюшкин весьма доволен наличием выравнивания в структурах и уже подсчитывал барыши, но оказался плох в математике.

Плюшкин полностью доверяет Вам в решении данной проблемы и даже готов переустановить свою операционную систему и поставить `IA-32/Linux`, `IA-32/Windows`, `x86_64/Linux` или `x86_64/Windows`.

Структура данных о крестьянине, предлагаемая Плюшкиным, выглядит следующим образом:

```c
struct peasant {
	char name[2]; // первые буквы имён крестьянина и его отца
	double usability; // КПД крестьянина (зачем он Чичикову - без комментариев)
	unsigned short age; // возраст крестьянина
	union {
		short long_code[5];
		long short_code[3];
	} id; // уникальный идентификатор крестьянина
	struct {
		signed char send;
		short solution;
		unsigned char asap;
	} filler; // какая-то бессмыслица, которую Плюшкину удалось внедрить в требования
}
```

В **единственной строке** ответа выпишите 3 целых числа - количество крестьян, данные о которых заполнят дискету (возможно, останутся пустые байты, но ещё одна запись в них не поместится) в следующих условиях:

- **A.** Порядок следования полей структуры соответствует предложенному Плюшкиным, и Вы выбираете операционную систему, позволяющую максимизировать размер структуры
- **B.** Чичиков требует использовать конкретную операционную систему (в которой размер предложенной Плюшкиным структуры минимален), однако порядок следования полей структуры по-прежнему определяется Плюшкиным
- **C.** Чичиков фиксирует не только операционную систему, но и меняет порядок полей (включая поля вложенных структур) так, чтобы структура занимала как можно меньше места

## Пример форматирования ответа:

```text
10 20 30
```

### Пояснение
- `A`: Берём `x86_64/Linux`: `long`/`double` - 8 байт + выравниваются по 8 байт, считаем [2(+6) + 8 + 2(+6) + max{10, 24} + (1(+1) + 2 + 1)(+1)] = 54, выравниваем по 8 = 56

- `B`: Берём `IA-32/Linux`: `long` - 4 байта, `double` - 8 байт, выравнивается по 4 байта, считаем [2(+2) + 8 + 2(+2) + max{10, 12} + (1(+1) + 2 + 1)(+1)] = 34, выравниваем по 4 = 36

- `C`: Берём `IA-32/Linux`, перекладываем структуру в минимальную (максимальные вперёд, минимальные назад), считаем [4 + max{10, 12} + (2 + 1 + 1) + 2 + 1] = 27, выравниваем по 4 = 28

Переложенная структура:
```c
struct peasant {
	double usability;
	union {
		short long_code[5];
		long short_code[3];
	} id;
	struct {
		short solution;
		signed char send;
		unsigned char asap;
	} filler;
	unsigned short age;
	char name[2];
}
```

4096 / 56 = A  
4096 / 36 = B  
4096 / 28 = C
